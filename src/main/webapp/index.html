Hello RESTful!

Для доступа к необходимому ресурсу приложения нужно отправить на сервер подходящий HTTP-запрос.
Поскольку в браузерной строке можно отправить только GET-запрос, рассмотрим несколько его примеров.

Получение списка всех сделок

Соответствующий GET-запрос: api/deals
где "api" - это точка входа в приложение, а "deals" - ресурс, содержащий инфорацию о сделках.

Получение определенной сделки

В этом случае к предыдущему запросу добавляется номер конкретной сделки: api/deals/{номер сделки}
Например, api/deals/345

Использование дополнительных ресурсов

Запрос может быть составным и включать доп.ресурсы, например: api/deals/agents/{имя агента}
В этом случае ресурс "agents" является дополнительным.
Аналогично, доп.ресурсами могут быть остальные параметры модели данных,
как в нашем случае дата сделки: api/deals/dates/{дата сделки} Например, api/deals/dates/2021-06-01

Комбинирование дополнительных ресурсов

Запрос может быть комбинацией доп.ресурсов.
Например: api/deals/agents/{имя агента}/dates/{дата сделки}
Результатом запроса будет сделка с конкретным агентом на определенную дату. Например, api/deals/agents/Alex/dates/2021-06-01

Для наглядности, рассмотрим пример получения списка сделок с конкретным агентом за некоторый период:
api/deals/agents/{имя агента}/startdate/{начало периода}/enddate/{конец периода}
api/deals/agents/Bob/startdate/2021-06-01/enddate/2021-06-06
